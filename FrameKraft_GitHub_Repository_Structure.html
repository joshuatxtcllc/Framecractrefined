🚀 FrameKraft Repository Structure

```

⚠️ CRITICAL: This Exact Structure Will Fix Your Deployment Issues

Copy this structure EXACTLY. Every file placement matters for successful deployment.

``` framekraft/ ← Your GitHub Repository Root ├── package.json ← ROOT: Your main package.json (CRITICAL!) ├── railway.toml ← Railway deployment config ├── vercel.json ← Vercel deployment config ├── netlify.toml ← Netlify deployment config ├── .replit ← Replit config (keep as is) ├── .gitignore ← Git ignore file ├── README.md ← Project documentation ├── DEPLOYMENT.md ← Your existing deployment guide ├── components.json ← UI components config ├── add_tax_exempt_column.sql ← Database migration ├── client/ ← Frontend React App │ ├── package.json ← Client dependencies │ ├── vite.config.js ← Vite build configuration │ ├── tailwind.config.js ← Tailwind CSS config │ ├── index.html ← Entry HTML file │ └── src/ ← React source code │ ├── main.tsx ← React entry point │ ├── App.tsx ← Main App component │ ├── components/ ← React components │ ├── pages/ ← Page components │ └── styles/ ← CSS/styling files ├── server/ ← Backend Express App │ ├── index.ts ← Main server entry point │ ├── routes/ ← API route handlers │ ├── middleware/ ← Express middleware │ ├── db/ ← Database configuration │ └── utils/ ← Utility functions ├── shared/ ← Shared code between client/server │ ├── types/ ← TypeScript type definitions │ ├── utils/ ← Shared utility functions │ └── constants/ ← Shared constants ├── attached_assets/ ← Static assets (images, docs, etc.) ├── dist/ ← Built production files (auto-generated) ├── node_modules/ ← Dependencies (auto-generated) └── .env.example ← Environment variables template
```

🎯 Step 1: Create These Files in Your Repository Root

Add these deployment configuration files to your main directory:

  • railway.toml - For Railway deployment
  • vercel.json - For Vercel deployment
  • netlify.toml - For Netlify deployment

Platform Support: Railway Vercel Netlify

🔧 Step 2: Your Root package.json MUST Have These Scripts

``` { “scripts”: { “dev”: “NODE_ENV=development tsx server/index.ts”, “build”: “vite build && esbuild server/index.ts –platform=node –packages=external –bundle –format=esm –outdir=dist”, “start”: “NODE_ENV=production node dist/index.js”, “check”: “tsc”, “db:push”: “drizzle-kit push” } }
```

✅ Why This Structure Works

  • Root package.json - Deployment platforms look here first
  • Build commands - Correctly builds both client (Vite) and server (esbuild)
  • Start command - Points to the compiled server in dist/
  • Config files - Tell each platform exactly how to deploy

🚀 Step 3: Deployment Process

  1. Push to GitHub with this exact structure
  2. Connect to Railway/Vercel/Netlify
  3. Platform reads config file (railway.toml, vercel.json, etc.)
  4. Runs npm run build - Creates production files
  5. Runs npm start - Starts your server
  6. Your app is LIVE! 🎉

🚨 Common Mistakes That Break Deployment

  • ❌ Missing root package.json
  • ❌ Wrong build scripts
  • ❌ No deployment config files
  • ❌ Incorrect file paths in scripts
  • ❌ Missing environment variables
